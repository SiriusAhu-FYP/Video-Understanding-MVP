# GameScene-Eval: v2 → v3 升级说明

> 基于 `v2/升级草稿.md` 整理，记录从 v2 到 GameScene-Eval (v3) 的全部变更和设计决策。

## 一、升级背景

基于 `QA/v2/02_实验设计自省.md` 和 `QA/v2/03_组会汇报指南.md` 的自省分析，v2 实验 (`experiments/v2-results/2026-03-16_18-09-38`) 存在以下核心不足：

1. **缺乏量化质量评估**：v2 仅测速度，无法回答「哪个模型理解得最好」
2. **提示词过于简陋**：`prompts/` 下的通用提示词缺乏任务针对性
3. **统计实践不够严格**：仅 3 次重复，无置信区间
4. **toolkit 耦合度高**：`toolkit/` 混入了 WSL 管理脚本，不可移植

本次升级围绕两大任务展开：**新实验设计** 和 **toolkit 转换**。

## 二、新实验设计

### 2.1 实验素材更新

| 变更 | 详情 |
|------|------|
| 放弃旧素材 | `assets_old/` 下的无描述素材不再使用 |
| 新增结构化 JSON | 每个媒体文件配套 `.json` 任务描述文件 |
| 统一字段顺序 | `id → type → game → image_file/video_file → requirement → ...` |
| 覆盖范围 | 4 张游戏截图（Minecraft ×2、Genshin Impact ×2）+ 2 个游戏录屏 |

每个 JSON 文件包含：
- 任务定义 (`task_definition`)
- 双模式提示词 (`prompts.A_description` / `prompts.B_assistant`)
- 参考答案 (`reference_answer`)
- 5 维度评分标准 (`scoring.dimensions`)
- LLM-Judge 评分提示词 (`grading_prompt_for_judge_model`)

### 2.2 综合实验设计 v3

详见 `blueprint/综合实验设计_v3.md`，核心改进：

| 维度 | v2 | v3 |
|------|----|----|
| 质量评估 | 无 | LLM-as-Judge (DeepSeek) 量化评分 |
| 实验数量 | 2 个 | 3 个（新增 benchmark_quality） |
| 重复次数 | 3 | 5（配 95% CI） |
| 提示词 | `prompts/` 文件 | JSON 内嵌任务专属提示词 |
| 统计指标 | mean ± std | mean ± std + 95% CI + 维度分解 |
| 工具包 | `toolkit/` | `ahu_paimon_toolkit` (git submodule) |

### 2.3 新增实验：benchmark_quality

- **入口**：`experiments/benchmark_quality/benchmark.py`
- **流程**：VLM 推理 → DeepSeek Judge 评分 → CSV + Markdown 报告
- **评分机制**：每维度 0-2 分，5 维度共 10 分
- **集成**：`run_benchmark.py` 新增 `benchmark_quality` 阶段

### 2.4 实验脚本更新

所有实验脚本已从 `toolkit.*` 迁移至 `ahu_paimon_toolkit.*`：

| 文件 | 变更 |
|------|------|
| `run_benchmark.py` | 导入迁移 + 新增 quality 阶段 + `--skip-quality` 参数 |
| `experiments/benchmark_speed/benchmark.py` | 导入迁移至 toolkit |
| `experiments/video_understanding/run_experiment.py` | 导入迁移至 toolkit |
| `config_benchmark.toml` | 新增 `[benchmark_quality]`，重复次数升至 5 |

### 2.5 提示词策略

| 变更 | 详情 |
|------|------|
| 删除 `prompts/` 目录 | `vlm_prompt.md` 和 `deepseek_prompt.md` 已删除 |
| 提示词嵌入 JSON | 每个素材的 JSON 内含 `A_description` 和 `B_assistant` 两种提示词 |
| Summarizer 默认提示词 | 视频理解的汇总提示词由 `ahu_paimon_toolkit.pipeline.summarizer` 内置管理 |
| Judge 提示词 | `grading_prompt_for_judge_model` 字段由 JSON 提供 |

## 三、Toolkit 转换

### 3.1 设计原则

- **解耦**：不依赖主项目的任何文件，可独立安装使用
- **模块化**：按功能域划分子包 (`capture/`, `vlm/`, `evaluation/`, `pipeline/`, `utils/`)
- **类型安全**：Pydantic 数据模型 + `py.typed` 标记
- **可测试**：所有公开 API 有配套的 mock 测试

### 3.2 新 toolkit 结构

```
ahu_paimon_toolkit/src/ahu_paimon_toolkit/
├── __init__.py
├── config.py           # ToolkitSettings (Pydantic)
├── models.py           # 共享数据模型 (KeyFrame, FrameDescription, JudgeScore, etc.)
├── capture/
│   ├── frame_diff.py   # MSE/SSIM 帧差算法
│   ├── window_capture.py # mss + pywin32 窗口截图
│   └── video_reader.py # OpenCV 视频帧提取
├── vlm/
│   ├── client.py       # AsyncVLMClient (OpenAI-compatible)
│   └── model_utils.py  # 模型检测、vision 判断、vLLM 就绪检查
├── evaluation/
│   ├── judge.py        # LLMJudge (DeepSeek API)
│   └── scoring.py      # ScoreAggregator (mean, stdev, CI)
├── pipeline/
│   ├── queue_manager.py # KeyFrameQueue (过期淘汰)
│   ├── summarizer.py    # DeepSeek 视频汇总
│   └── video_understanding.py # 完整流水线封装
└── utils/
    ├── image.py         # base64 编码、MIME 检测、批量加载
    ├── gpu.py           # GPU 信息查询
    └── visualization.py # matplotlib/seaborn 图表
```

### 3.3 旧 toolkit 处理

| 原文件 | 去向 |
|--------|------|
| `toolkit/common.py` | 拆分至 `utils/image.py`, `utils/gpu.py`, `vlm/model_utils.py` |
| `toolkit/visualization.py` | 迁移至 `utils/visualization.py` |
| `toolkit/vllm_manager.py` | 移至 `scripts/vllm_manager.py`（WSL 管理不属于通用 toolkit） |

### 3.4 Git Submodule 管理

- `ahu_paimon_toolkit` 作为独立 git 仓库，通过 `.gitmodules` 声明
- 主项目通过 `pyproject.toml` 的 `[tool.uv.sources]` 声明 editable 路径依赖
- toolkit 内部变更在子模块中独立提交，主项目更新子模块引用
