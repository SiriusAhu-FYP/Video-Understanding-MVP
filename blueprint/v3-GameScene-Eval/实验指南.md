# GameScene-Eval 实验指南

> 本文档指导实验者从零开始运行 GameScene-Eval (v3) 基准测试。

## 一、环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11 + WSL2 (Ubuntu) |
| GPU | NVIDIA GPU, >= 12GB VRAM (推荐 16GB) |
| Python | >= 3.12 |
| 包管理器 | [uv](https://docs.astral.sh/uv/) |
| vLLM | 在 WSL2 中安装 (>= 0.17.0) |
| 播放器 | PotPlayer（video_understanding 实验需要） |
| API Key | DeepSeek API Key（benchmark_quality + 视频汇总需要） |

## 二、初始设置

### 2.1 克隆并安装

```bash
git clone --recurse-submodules <repo-url>
cd Video-Understanding-MVP
uv sync
```

如果已经克隆但缺少子模块：

```bash
git submodule update --init --recursive
uv sync
```

### 2.2 配置 `.env`

在项目根目录创建 `.env` 文件：

```env
DEEPSEEK_API_KEY=sk-your-deepseek-api-key
DEEPSEEK_API_BASE_URL=https://api.deepseek.com
PLAYER_EXE_PATH=D:\Programs\PotPlayer\PotPlayerMini64.exe
```

- `DEEPSEEK_API_KEY`：必需，用于 LLM-as-Judge 评分和视频汇总
- `PLAYER_EXE_PATH`：仅 video_understanding 实验需要

### 2.3 WSL 中准备 vLLM

确保模型权重已缓存在 WSL 的 `~/.cache/huggingface/hub/` 目录中。

WSL 环境需要额外安装：
- `timm`（DeepSeek-VL2 依赖）
- `num2words`（SmolVLM2 依赖）

### 2.4 验证安装

```bash
uv run pytest tests/test_toolkit.py -v
```

24 个测试应全部通过。

## 三、实验配置

配置文件：`config_benchmark.toml`

### 关键配置项

```toml
[models]
list = [
    "Qwen/Qwen3.5-2B",
    "Qwen/Qwen3.5-0.8B",
    # ... 共 8 个模型
]

[benchmark_speed]
enabled = true
runs = 5          # 每场景重复次数（不含 1 次预热）

[benchmark_quality]
enabled = true
runs = 5          # 每 (素材 x 提示词) 重复次数
judge_delay_s = 1.0  # Judge API 调用间隔

[video_understanding]
enabled = true
runs = 5          # 每视频重复次数
```

如需调试或快速测试，可以：
- 减少 `models.list` 中的模型数量
- 将 `runs` 改为 `1`
- 将不需要的实验的 `enabled` 设为 `false`

## 四、运行实验

### 4.1 完整编排（推荐）

```bash
uv run run_benchmark.py
```

完整流程会自动：
1. 逐个启动 vLLM 加载模型
2. 运行 benchmark_speed（推理速度）
3. 运行 benchmark_quality（LLM-as-Judge 质量评分）
4. 运行 video_understanding（视频理解流水线）
5. 生成对比报告和可视化图表

### 4.2 常用参数

```bash
# 跳过某个实验
uv run run_benchmark.py --skip-speed
uv run run_benchmark.py --skip-quality
uv run run_benchmark.py --skip-video

# 仅测试特定模型
uv run run_benchmark.py --models "Qwen/Qwen3.5-2B" "Qwen/Qwen3-VL-2B-Instruct"

# 使用自定义配置
uv run run_benchmark.py --config my_config.toml

# 跳过预飞行验证（模型已知可用时）
uv run run_benchmark.py --skip-preflight

# 跳过 GPU 显存探测（使用默认值）
uv run run_benchmark.py --skip-memory-probe
```

### 4.3 单独运行各实验

如果已经手动启动了 vLLM 服务，可以单独运行：

```bash
# 推理速度
uv run experiments/benchmark_speed/benchmark.py --base-url http://172.x.x.x:8000/v1

# 质量评估
uv run experiments/benchmark_quality/benchmark.py --base-url http://172.x.x.x:8000/v1

# 视频理解（需要 PotPlayer 已安装）
uv run experiments/video_understanding/run_experiment.py
```

## 六、常见问题

### vLLM 启动失败

- 检查 GPU 显存是否足够（`nvidia-smi`）
- 尝试增大 `gpu_memory_utilization`（配置文件中默认 0.7）
- 确认 `--enforce-eager` 参数已启用（减少显存开销）

### DeepSeek API 报错

- 检查 `.env` 中的 `DEEPSEEK_API_KEY` 是否正确
- DeepSeek 有 QPM 限制，可增大 `judge_delay_s`

### PotPlayer 相关问题

- 确认 `PLAYER_EXE_PATH` 路径正确
- PotPlayer 窗口不能被最小化（需要可见才能截图）

### 模型推理返回空响应

- 某些模型在特定 prompt 下可能返回极短或空响应
- 预飞行检查 (`run_preflight`) 会自动过滤这类模型
